# App Settings

Здесь собраны конфигурационные параметры, скрытые от пользователя в UI, но полезные для разработки и ручной настройки системы.

### 1. Ограничение вывода строк таблицы (maxRowsToDisplay)

**Где находится:** `frontend/src/store/queryStore.ts` (стейт `maxRowsToDisplay`)
**По умолчанию:** `100`

**Зачем нужно:** 
Чтобы при запросе вида `SELECT * FROM big_table` браузер не умирал, пытаясь отрендерить 100 000 строк в UI (вкладка Result). Бэкенд по-прежнему может вернуть много строк (если мы не ограничили его там), но интерфейс отрисует лишь указанное число. Если число строк превышает `maxRowsToDisplay`, таблица показывает предупреждение, что данные обрезаны (`isTruncated = true`).

### 2. EXPLAIN опции по умолчанию (analyze, buffers)

**Где находится:** `frontend/src/store/explainStore.ts` (объект `options: { analyze: true, buffers: true }`)
**По умолчанию:** обе опции включены (`true`).

**Зачем нужно:**
В ранних версиях спецификации предлагалось выносить переключатели `[ANALYZE]` и `[BUFFERS]` в UI, чтобы не замедлять запросы. Но так как это тренажер для локального (одного) пользователя, мы не боимся дополнительных накладных расходов. Поэтому кнопки из UI были убраны, а флаги жестко зашиты в `true`. Это значит, что бэкенд *всегда* будет вызывать `EXPLAIN (ANALYZE, BUFFERS, FORMAT JSON)`.
